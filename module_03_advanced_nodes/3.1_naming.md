## Fully-Qualified Names

Every resource on the ROS2 graph — nodes, topics, services, actions — has a
**fully-qualified name**: a path-like string that uniquely identifies it, always
starting with `/`.

```
/circle_publisher          ← a node
/turtle1/cmd_vel           ← a topic
/turtle1/set_pen           ← a service
/turtle1/rotate_absolute   ← an action
```

When you run `ros2 topic list` or `ros2 node list`, you always see fully-qualified
names. The leading `/` is part of the name, not just a display convention.

---

## Name Forms in Code

In rclpy, the string you pass to `create_publisher`, `create_subscription`,
`create_client`, or `create_service` determines how the name is resolved. There are
three forms:

| Form | Example in code | Resolved name (no namespace) | Resolved name (`__ns:=/robot1`) |
|---|---|---|---|
| Absolute | `'/turtle1/cmd_vel'` | `/turtle1/cmd_vel` | `/turtle1/cmd_vel` |
| Relative | `'turtle1/cmd_vel'` | `/turtle1/cmd_vel` | `/robot1/turtle1/cmd_vel` |
| Private | `'~/cmd_vel'` | `/<node_name>/cmd_vel` | `/robot1/<node_name>/cmd_vel` |

In Module 2, all turtle-specific topics and services were written as **absolute names**
(`'/turtle1/cmd_vel'`, `'/turtle1/pose'`). The leading `/` makes the target explicit —
the node always talks to turtle1 no matter what. That clarity is useful when you are
learning, but it also locks the node to a single turtle permanently.

---

## The Limitation of Absolute Names

**Terminal 1** — start turtlesim:

```bash
ros2 run turtlesim turtlesim_node
```

**Terminal 2** — run `circle_publisher` under a namespace:

```bash
ros2 run turtlesim_controller circle_publisher --ros-args -r __ns:=/robot1
```

Check the topic list:

```bash
ros2 topic list
```

You will see `/robot1/turtle1/cmd_vel` — the node is running in namespace `/robot1`, so
all relative names would be prefixed with `/robot1/`. But the turtle is not moving.

Now run:

```bash
ros2 node info /robot1/circle_publisher
```

Look at the Publishers section. You will see `/turtle1/cmd_vel` — without the
`/robot1/` prefix. The absolute name `'/turtle1/cmd_vel'` in the code bypasses the
namespace completely. The publisher connects to `/turtle1/cmd_vel` (turtle1's topic),
not `/robot1/turtle1/cmd_vel`. The namespace had no effect.

Press `Ctrl+C` to stop `circle_publisher`.

## Remapping

Namespacing redirects every relative name at once. Remapping gives you finer control:
you redirect a specific name to a specific target.

```bash
--ros-args -r source_name:=target_name
```

`source_name` is the relative name as the node knows it. `target_name` is the actual
fully-qualified name it should reach. You can also remap the node's own name:

```bash
--ros-args -r __node:=circle_publisher_2
```

---

## Step 4 — Remap to a Different Turtle

Keep turtlesim running. Spawn a second turtle:

```bash
ros2 service call /spawn turtlesim/srv/Spawn \
  "{x: 3.0, y: 3.0, theta: 0.0, name: 'turtle2'}"
```

Run `circle_publisher` again, directed at turtle2 via remapping:

```bash
ros2 run turtlesim_controller circle_publisher \
  --ros-args -r turtle1/cmd_vel:=turtle2/cmd_vel -r __node:=circle_publisher_2
```

Turtle2 circles. Turtle1 stays still.

In a separate terminal, run the original instance for turtle1:

```bash
ros2 run turtlesim_controller circle_publisher
```

Both turtles circle simultaneously. The source code is identical — only the launch
arguments differ.

Press `Ctrl+C` in both terminals.

## Exercise — Convert `turtle_controller.py`

`turtle_controller.py` was written in Module 2 with absolute names. It controls turtle1
and can never be redirected. Apply the same fix: change the turtle-specific names to
relative so the node can be remapped to any turtle at launch.

Here is the starting code — the 2.7 answer key as written:

```python
import rclpy
from rclpy.node import Node
from geometry_msgs.msg import Twist, Point
from turtlesim.msg import Pose
from turtlesim_controller.control_law import compute_go_to_goal_control


class TurtleController(Node):
    def __init__(self):
        super().__init__('turtle_controller')
        self.current_pose = None
        self.current_goal = None
        self.pub = self.create_publisher(Twist, '/turtle1/cmd_vel', 10)
        self.create_subscription(Pose, '/turtle1/pose', self.pose_callback, 10)
        self.create_subscription(Point, 'goal', self.goal_callback, 10)
        self.create_timer(0.1, self.control_loop)

    def pose_callback(self, msg: Pose):
        self.current_pose = [msg.x, msg.y, msg.theta]

    def goal_callback(self, msg: Point):
        self.current_goal = [msg.x, msg.y]

    def control_loop(self):
        if self.current_pose is None or self.current_goal is None:
            return
        v, w, d = compute_go_to_goal_control(self.current_pose, self.current_goal)
        cmd = Twist()
        if d < 0.1:
            self.pub.publish(cmd)
            return
        cmd.linear.x = v
        cmd.angular.z = w
        self.pub.publish(cmd)


def main(args=None):
    rclpy.init(args=args)
    node = TurtleController()
    try:
        rclpy.spin(node)
    except KeyboardInterrupt:
        pass
    finally:
        node.destroy_node()
        if rclpy.ok():
            rclpy.shutdown()


if __name__ == '__main__':
    main()
```

## Answer Key

The only lines that change in `turtle_controller.py`:

```python
self.pub = self.create_publisher(Twist, 'turtle1/cmd_vel', 10)
self.create_subscription(Pose, 'turtle1/pose', self.pose_callback, 10)
```

`'goal'` is not changed because it is a shared input — both controller instances
read from the same `/goal` topic, which is correct.

Why absolute names block remapping: the remap flag `turtle1/cmd_vel:=turtle2/cmd_vel`
intercepts the relative name `turtle1/cmd_vel` before it is resolved. An absolute name
`/turtle1/cmd_vel` is resolved before the remap table is consulted — it bypasses the
remap and always reaches turtle1.

---

## Summary

| Concept | Key point |
|---|---|
| Fully-qualified name | Starts with `/`; shown by `ros2 topic list` / `ros2 node list` |
| Absolute `'/foo'` | Always `/foo`; ignores namespace and remap |
| Relative `'foo'` | Prepend namespace; intercepted by remap table |
| Private `'~/foo'` | Prepend `/<node_name>` then namespace |
| `--ros-args -r __ns:=/ns` | Sets namespace; prefixes all relative names |
| `--ros-args -r src:=dst` | Remaps `src` (relative name) to `dst` at launch |
| `--ros-args -r __node:=name` | Renames the node |
