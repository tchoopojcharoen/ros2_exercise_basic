## Services: Request and Response

A **service** is a synchronous, point-to-point communication pattern: one node sends a request and waits for a reply. Unlike a topic, which streams continuously, a service call completes once and returns a result.

```
client ──► request ──► server
client ◄── response ◄── server
```

Every service has a **type** that defines the request and response fields, separated by `---`:

```
<request fields>
---
<response fields>
```

```bash
ros2 service type /turtle1/set_pen
```

Output:
```
turtlesim/srv/SetPen
```

Inspect the fields:

```bash
ros2 interface show turtlesim/srv/SetPen
```

Output:
```
uint8 r
uint8 g
uint8 b
uint8 width
uint8 off
---

```

The response is empty — `set_pen` configures state on the server and returns nothing. The `off` field is `1` to lift the pen (no trail) and `0` to draw.

## Creating a Service Client

Three steps per call:

1. **Create the client** in `__init__`, bound to a service name and type:
   ```python
   self.pen_client = self.create_client(SetPen, 'turtle1/set_pen')
   ```

2. **Build a request and send it:**
   ```python
   req = SetPen.Request()
   req.r, req.g, req.b, req.width, req.off = 255, 0, 0, 3, 0
   future = self.pen_client.call_async(req)
   ```

3. **Wait for the response** — but only when you are outside a callback:
   ```python
   rclpy.spin_until_future_complete(self, future)
   ```

`call_async` returns a `Future` immediately. `spin_until_future_complete` drives the event loop until that future resolves. This is valid before `rclpy.spin()` starts, because no executor is running yet. From inside a callback, the executor is already running — calling `spin_until_future_complete` there would deadlock. That situation is covered in 3.6.

From inside a callback, you can still fire a service call — you just cannot wait for it:

```python
self.pen_client.call_async(req)   # fire and forget — valid in callbacks
```

Since `set_pen` returns an empty response, there is nothing to wait for. The executor processes the response automatically when the callback returns.

Extend `timer_callback` to count ticks and change color at each revolution boundary. `call_async` is used here — fire and forget, no waiting:

```python
def timer_callback(self):
    radius = self.get_parameter('radius').value
    w = self.get_parameter('angular_speed').value
    msg = Twist()
    msg.linear.x = radius * w
    msg.angular.z = w
    self.pub.publish(msg)

    self.tick_count += 1
    ticks_per_revolution = int(2 * math.pi / (w * 0.1))
    if self.tick_count % ticks_per_revolution == 0:
        self.revolution += 1
        r, g, b = COLORS[self.revolution % len(COLORS)]
        req = SetPen.Request()
        req.r, req.g, req.b, req.width, req.off = r, g, b, 3, 0
        self.pen_client.call_async(req)
```

The complete file after all changes:

```python
import math

import rclpy
from rclpy.node import Node
from geometry_msgs.msg import Twist
from rcl_interfaces.msg import SetParametersResult
from turtlesim.srv import SetPen

COLORS = [(255, 0, 0), (0, 200, 0), (0, 0, 255)]


class CirclePublisher(Node):
    def __init__(self):
        super().__init__('circle_publisher')
        self.declare_parameter('radius', 1.0)
        self.declare_parameter('angular_speed', 1.0)
        self.pub = self.create_publisher(Twist, 'turtle1/cmd_vel', 10)
        self.pen_client = self.create_client(SetPen, 'turtle1/set_pen')
        self.tick_count = 0
        self.revolution = 0
        self.create_timer(0.1, self.timer_callback)
        self.add_on_set_parameters_callback(self.validate_params)

    def configure_pen(self, r, g, b, width):
        self.pen_client.wait_for_service()
        req = SetPen.Request()
        req.r = r
        req.g = g
        req.b = b
        req.width = width
        req.off = 0
        future = self.pen_client.call_async(req)
        rclpy.spin_until_future_complete(self, future)

    def timer_callback(self):
        radius = self.get_parameter('radius').value
        w = self.get_parameter('angular_speed').value
        msg = Twist()
        msg.linear.x = radius * w
        msg.angular.z = w
        self.pub.publish(msg)

        self.tick_count += 1
        ticks_per_revolution = int(2 * math.pi / (w * 0.1))
        if self.tick_count % ticks_per_revolution == 0:
            self.revolution += 1
            r, g, b = COLORS[self.revolution % len(COLORS)]
            req = SetPen.Request()
            req.r, req.g, req.b, req.width, req.off = r, g, b, 3, 0
            self.pen_client.call_async(req)

    def validate_params(self, params):
        for p in params:
            if p.name in ('radius', 'angular_speed') and p.value <= 0:
                return SetParametersResult(
                    successful=False,
                    reason=f'{p.name} must be positive'
                )
        return SetParametersResult(successful=True)


def main(args=None):
    rclpy.init(args=args)
    node = CirclePublisher()
    node.configure_pen(r=255, g=0, b=0, width=3)
    try:
        rclpy.spin(node)
    except KeyboardInterrupt:
        pass
    finally:
        node.destroy_node()
        if rclpy.ok():
            rclpy.shutdown()


if __name__ == '__main__':
    main()
```

No new entry point. No new dependency — `turtlesim` is already in `package.xml`.

## Answer Key

```python
import math
import random

import rclpy
from rclpy.node import Node
from geometry_msgs.msg import Twist
from rcl_interfaces.msg import SetParametersResult
from turtlesim.srv import SetPen, TeleportAbsolute

COLORS = [(255, 0, 0), (0, 200, 0), (0, 0, 255)]


class CirclePublisher(Node):
    def __init__(self):
        super().__init__('circle_publisher')
        self.declare_parameter('radius', 1.0)
        self.declare_parameter('angular_speed', 1.0)
        self.pub = self.create_publisher(Twist, 'turtle1/cmd_vel', 10)
        self.pen_client = self.create_client(SetPen, 'turtle1/set_pen')
        self.teleport_client = self.create_client(TeleportAbsolute, 'turtle1/teleport_absolute')
        self.tick_count = 0
        self.revolution = 0
        self.create_timer(0.1, self.timer_callback)
        self.add_on_set_parameters_callback(self.validate_params)

    def configure_pen(self, r, g, b, width):
        self.pen_client.wait_for_service()
        req = SetPen.Request()
        req.r = r
        req.g = g
        req.b = b
        req.width = width
        req.off = 0
        future = self.pen_client.call_async(req)
        rclpy.spin_until_future_complete(self, future)

    def timer_callback(self):
        radius = self.get_parameter('radius').value
        w = self.get_parameter('angular_speed').value
        msg = Twist()
        msg.linear.x = radius * w
        msg.angular.z = w
        self.pub.publish(msg)

        self.tick_count += 1

        ticks_per_revolution = int(2 * math.pi / (w * 0.1))
        if self.tick_count % ticks_per_revolution == 0:
            self.revolution += 1
            r, g, b = COLORS[self.revolution % len(COLORS)]
            req = SetPen.Request()
            req.r, req.g, req.b, req.width, req.off = r, g, b, 3, 0
            self.pen_client.call_async(req)

            treq = TeleportAbsolute.Request()
            treq.x = random.uniform(0.0, 11.88)
            treq.y = random.uniform(0.0, 11.88)
            treq.theta = 0.0
            self.teleport_client.call_async(treq)

    def validate_params(self, params):
        for p in params:
            if p.name in ('radius', 'angular_speed') and p.value <= 0:
                return SetParametersResult(
                    successful=False,
                    reason=f'{p.name} must be positive'
                )
        return SetParametersResult(successful=True)


def main(args=None):
    rclpy.init(args=args)
    node = CirclePublisher()
    node.configure_pen(r=255, g=0, b=0, width=3)
    node.teleport_client.wait_for_service()
    try:
        rclpy.spin(node)
    except KeyboardInterrupt:
        pass
    finally:
        node.destroy_node()
        if rclpy.ok():
            rclpy.shutdown()


if __name__ == '__main__':
    main()
```

The color change and the teleport both fire inside the same revolution boundary block — two separate `call_async` calls, both fire-and-forget. `spin_until_future_complete` is reserved for `configure_pen`, which is called from `main()` before `rclpy.spin()` starts. A node can hold any number of service clients; each is just a handle registered in `__init__`.

## Summary

| Concept | Key point |
|---|---|
| `ros2 service list` | Lists all services currently on the ROS2 graph |
| `ros2 service type /name` | Shows the service type (e.g. `turtlesim/srv/SetPen`) |
| `ros2 interface show pkg/srv/Type` | Shows request and response fields; `---` separates them |
| `create_client(SrvType, '/name')` | Creates a client handle in `__init__`; reuse across calls |
| `wait_for_service()` | Blocks until the server is reachable; call once before the first use |
| `call_async(req)` | Sends the request; returns a Future immediately |
| `spin_until_future_complete(self, future)` | Valid in `main()` before `rclpy.spin()` starts; deadlocks inside a callback |
| `call_async` without waiting | Valid inside a callback — fire and forget; use when the response is empty or not needed |
