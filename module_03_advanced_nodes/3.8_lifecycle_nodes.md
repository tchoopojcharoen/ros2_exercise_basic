## Why Lifecycle?

A standard `Node` is binary: it is either running or it is not. When you press `Ctrl+C`, the process exits and whatever the robot was doing simply stops ŌĆö the last velocity command stays active, the turtle drifts.

In production robot systems this is not acceptable. Two problems come up repeatedly:

**Unsafe stop.** When `circle_publisher` is killed, no zero-velocity command is sent. The turtle keeps its last `cmd_vel` and continues drawing until turtlesim's own damping stops it. On a physical robot, the motors keep running at the last commanded speed until the next command arrives ŌĆö or until something collides.

**Uncoordinated startup.** In a three-node system, a controller might start publishing `cmd_vel` before `turtlesim_node` has finished initialising, before the first pose message has arrived, or before an operator has confirmed the system is safe to move.

`LifecycleNode` solves both by adding an explicit state machine between "created" and "active". You drive the node through its states intentionally ŌĆö configure first, then activate ŌĆö and each transition has a defined callback where you do the right thing.

## The Lifecycle State Machine

A lifecycle node moves through five primary states:

```
              configure              activate
[Unconfigured] ŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌ¢║ [Inactive] ŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌ¢║ [Active]
      Ō¢▓                    Ōöé                       Ōöé
      Ōöé              cleanupŌöé               deactivateŌöé
      ŌööŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöśŌŚäŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöś

Any state ŌöĆŌöĆshutdownŌöĆŌöĆŌ¢║ [Finalized]
Any callback failure ŌöĆŌöĆŌ¢║ [ErrorProcessing]
```

| State | Meaning |
|---|---|
| `Unconfigured` | Node is created but has no ROS entities ŌĆö no publishers, subscribers, or timers |
| `Inactive` | ROS entities exist and are ready, but the node is not publishing |
| `Active` | Node is fully operational ŌĆö processing data and publishing outputs |
| `Finalized` | Node is shutting down; resources are being released |
| `ErrorProcessing` | A transition callback returned `FAILURE`; the node tries to recover |

Each arrow is a **transition** you trigger externally ŌĆö from the CLI, a launch file, or another node. When you trigger `configure`, rclpy calls your `on_configure` method. Your callback does the work and returns `TransitionCallbackReturn.SUCCESS` to confirm the transition, or `.FAILURE` to abort it.

## The Callback API

Inherit from `LifecycleNode` and override the callbacks you need:

```python
from rclpy.lifecycle import LifecycleNode, TransitionCallbackReturn
```

| Transition | Callback | Typical contents |
|---|---|---|
| `configure` | `on_configure(state)` | Validate params; create publishers, subscriptions, service clients, timer |
| `activate` | `on_activate(state)` | Set `enabled = True`; fire any one-time setup calls |
| `deactivate` | `on_deactivate(state)` | Clear flag; publish zero velocity |
| `cleanup` | `on_cleanup(state)` | Destroy ROS entities; reset data attributes |
| `shutdown` | `on_shutdown(state)` | Publish zero velocity as a safety net |
| (automatic) | `on_error(state)` | Publish zero velocity; return `SUCCESS` to recover to Unconfigured |

**Every callback must return** `TransitionCallbackReturn.SUCCESS` or `.FAILURE`. There is no response object to populate ŌĆö just return the enum.

**What belongs where:**
- `__init__`: declare parameters, register parameter callbacks, set entity handles to `None` ŌĆö **no `create_publisher`, `create_subscription`, `create_client`, or `create_timer`**
- `on_configure`: call all the `create_*` methods
- `on_activate`: flip the enable flag and fire one-time async setup calls

**One important shift from plain nodes:** initialization code that previously ran before `rclpy.spin()` in `main()` ŌĆö such as calling `spin_until_future_complete` ŌĆö cannot run inside a lifecycle callback, because spin is already running by the time callbacks are invoked. Use `call_async` instead. Fire-and-forget is sufficient for setup calls like setting the pen.

### The Guard Pattern

The timer is created in `on_configure` and fires continuously from that point on. But the node should not publish until it is active. The standard pattern:

```python
# in __init__
self.enabled = False

# in on_activate
self.enabled = True

# in on_deactivate
self.enabled = False

# in timer_callback
def timer_callback(self):
    if not self.enabled:
        return
    # ... publish only when active
```

The timer fires in every state after configure, but the callback returns immediately unless `enabled` is `True`.

## Worked Example ŌĆö `circle_lf_publisher.py`

`circle_lf_publisher` is the lifecycle version of `circle_publisher` from 3.5. The behaviour is identical ŌĆö the turtle draws color-cycling circles with a configurable radius and speed ŌĆö but the node is now manageable: you can configure it, activate it, and deactivate it without restarting the process.

Create `circle_lf_publisher.py` in `~/tutorial_ws/src/turtlesim_controller/turtlesim_controller/`.

### Step 1 ŌĆö Imports and constants

```python
import math
import random

import rclpy
from rclpy.lifecycle import LifecycleNode, TransitionCallbackReturn
from geometry_msgs.msg import Twist
from rcl_interfaces.msg import SetParametersResult
from std_srvs.srv import SetBool
from turtlesim.srv import SetPen, TeleportAbsolute

COLORS = [(255, 0, 0), (0, 200, 0), (0, 0, 255)]
```

The only change from `circle_publisher` is the import line: `LifecycleNode` and `TransitionCallbackReturn` replace `Node`.

### Step 2 ŌĆö `__init__`

```python
class CircleLfPublisher(LifecycleNode):
    def __init__(self):
        super().__init__('circle_lf_publisher')
        self.declare_parameter('radius', 1.0)
        self.declare_parameter('angular_speed', 1.0)
        self.pub = None
        self.pen_client = None
        self.teleport_client = None
        self.enabled = False
        self.tick_count = 0
        self.revolution = 0
        self.add_on_set_parameters_callback(self.validate_params)
```

`__init__` is for declaring parameters and initialising Python attributes ŌĆö nothing else. All the `create_*` calls are gone. Every ROS entity handle is set to `None` so the guard checks in `on_deactivate` and `on_shutdown` are safe before configure is ever called.

### Step 3 ŌĆö `on_configure`

```python
    def on_configure(self, state):
        self.pub = self.create_publisher(Twist, 'turtle1/cmd_vel', 10)
        self.pen_client = self.create_client(SetPen, 'turtle1/set_pen')
        self.teleport_client = self.create_client(
            TeleportAbsolute, 'turtle1/teleport_absolute'
        )
        self.create_service(SetBool, 'thick_pen', self.thick_pen_callback)
        self.create_timer(0.1, self.timer_callback)
        self.get_logger().info('Configured.')
        return TransitionCallbackReturn.SUCCESS
```

This is where all `create_*` calls live. The service server (`thick_pen`) and service clients (`set_pen`, `teleport_absolute`) are created here too ŌĆö not just publishers and timers. After `on_configure`, the topic interfaces and services exist and are ready; the node is `Inactive`.

### Step 4 ŌĆö `on_activate`

```python
    def on_activate(self, state):
        self.enabled = True
        if self.pen_client.service_is_ready():
            req = SetPen.Request()
            req.r, req.g, req.b, req.width, req.off = 255, 0, 0, 3, 0
            self.pen_client.call_async(req)
        self.get_logger().info('Activated.')
        return TransitionCallbackReturn.SUCCESS
```

Two things happen here. The enable flag is set. Then the initial pen color is configured ŌĆö this is the initialization that previously lived in `main()` before `rclpy.spin()`. Because spin is already running when `on_activate` is called, `spin_until_future_complete` would deadlock; `call_async` without waiting is the correct replacement.

`service_is_ready()` avoids calling into a service that has not yet appeared. If turtlesim is not running, the initialization is silently skipped ŌĆö the node activates regardless, and the pen keeps whatever color it was before.

### Step 5 ŌĆö `on_deactivate`

```python
    def on_deactivate(self, state):
        self.enabled = False
        if self.pub:
            self.pub.publish(Twist())
        self.get_logger().info('Deactivated ŌĆö published zero velocity.')
        return TransitionCallbackReturn.SUCCESS
```

Clearing the flag alone is not enough ŌĆö the last published `cmd_vel` would keep the turtle moving. Publishing a zero `Twist()` stops it explicitly. The `if self.pub` guard is safe even if `on_configure` was somehow never called.

### Step 6 ŌĆö `on_shutdown`

```python
    def on_shutdown(self, state):
        self.enabled = False
        if self.pub:
            self.pub.publish(Twist())
        return TransitionCallbackReturn.SUCCESS
```

`on_shutdown` is the last-resort safety net. Any path to shutdown ŌĆö clean or not ŌĆö passes through here.

### Step 7 ŌĆö `timer_callback` and the other methods

The remaining methods are unchanged from `circle_publisher`:

```python
    def thick_pen_callback(self, request, response):
        req = SetPen.Request()
        req.width = 5 if request.data else 1
        self.pen_client.call_async(req)
        response.success = True
        response.message = f'pen width: {req.width}'
        return response

    def timer_callback(self):
        if not self.enabled:
            return

        radius = self.get_parameter('radius').value
        w = self.get_parameter('angular_speed').value
        msg = Twist()
        msg.linear.x = radius * w
        msg.angular.z = w
        self.pub.publish(msg)

        self.tick_count += 1
        ticks_per_revolution = int(2 * math.pi / (w * 0.1))
        if self.tick_count % ticks_per_revolution == 0:
            self.revolution += 1
            r, g, b = COLORS[self.revolution % len(COLORS)]
            req = SetPen.Request()
            req.r, req.g, req.b, req.width, req.off = r, g, b, 3, 0
            self.pen_client.call_async(req)

            treq = TeleportAbsolute.Request()
            treq.x = random.uniform(0.0, 11.88)
            treq.y = random.uniform(0.0, 11.88)
            treq.theta = 0.0
            self.teleport_client.call_async(treq)

    def validate_params(self, params):
        for p in params:
            if p.name in ('radius', 'angular_speed') and p.value <= 0:
                return SetParametersResult(
                    successful=False,
                    reason=f'{p.name} must be positive'
                )
        return SetParametersResult(successful=True)
```

The only change in `timer_callback` is the guard at the top: `if not self.enabled: return`. The rest is identical to the 3.5 version. The timer fires in `Inactive` too, but the guard makes it a no-op.

### Step 8 ŌĆö `main()`

```python
def main(args=None):
    rclpy.init(args=args)
    node = CircleLfPublisher()
    try:
        rclpy.spin(node)
    except KeyboardInterrupt:
        pass
    finally:
        node.destroy_node()
        if rclpy.ok():
            rclpy.shutdown()


if __name__ == '__main__':
    main()
```

The `configure_pen` call and `wait_for_service` that previously appeared in `main()` before `rclpy.spin()` are gone ŌĆö that logic moved into `on_activate`. `main()` is now the standard three-line pattern.

## Exercise

Write `turtle_lf_controller.py` ŌĆö a lifecycle go-to-goal controller for turtlesim. Use the code below as your starting point.

**Description**

`turtle_lf_controller` navigates the turtle toward a goal point published on the `goal` topic. It supports `Kv`, `Kw`, and `tolerance` as dynamically reconfigurable ROS parameters, and accepts a `--frequency` argument (default 10.0 Hz) that sets the control loop rate.

The node is lifecycle-managed. Before it is configured, it has no publisher, subscriptions, or timer ŌĆö the turtle is unaffected by its presence. Once configured and activated, the control loop runs and the turtle navigates. When deactivated, the turtle stops immediately; the control loop pauses but the node stays alive. Reactivating resumes navigation toward the same goal. No `controller_enabled` service exists ŌĆö enable and disable are handled by the `activate` and `deactivate` transitions.

**Requirements**

- `Unconfigured`: node starts in this state; no ROS entities exist
- `Inactive`: publisher, subscriptions, and timer exist; control loop does not publish
- `Active`: control loop runs; turtle navigates toward the most recent goal
- `deactivate` ŌåÆ turtle stops immediately (a zero-velocity command is published at the moment of deactivation, not on the next timer tick)
- `shutdown` ŌåÆ turtle stops (zero-velocity command published)
- No `/controller_enabled` service ŌĆö `ros2 service list` must not show it
- All parameters (`Kv`, `Kw`, `tolerance`) must remain positive; reject negative or zero values

**Starting code:**

```python
import argparse

import rclpy
from rclpy.node import Node
from rclpy.qos import QoSPresetProfiles
from geometry_msgs.msg import Twist, Point
from turtlesim.msg import Pose
from rcl_interfaces.msg import SetParametersResult
from std_srvs.srv import SetBool
from turtlesim_controller.control_law import compute_go_to_goal_control


class TurtleController(Node):
    def __init__(self, frequency=10.0):
        super().__init__('turtle_controller')
        self.current_pose = None
        self.current_goal = None
        self.enabled = True
        self.declare_parameter('Kv', 1.0)
        self.declare_parameter('Kw', 5.0)
        self.declare_parameter('tolerance', 0.1)
        self.pub = self.create_publisher(Twist, 'turtle1/cmd_vel', 10)
        self.create_subscription(
            Pose, 'turtle1/pose', self.pose_callback,
            QoSPresetProfiles.SENSOR_DATA.value
        )
        self.create_subscription(Point, 'goal', self.goal_callback, 10)
        self.create_timer(1.0 / frequency, self.control_loop)
        self.add_on_set_parameters_callback(self.validate_params)
        self.create_service(SetBool, 'controller_enabled', self.controller_enabled_callback)
        self.get_logger().info(f'Control loop frequency: {frequency} Hz')

    def controller_enabled_callback(self, request, response):
        self.enabled = request.data
        response.success = True
        response.message = 'controller enabled' if request.data else 'controller disabled'
        return response

    def pose_callback(self, msg):
        self.current_pose = [msg.x, msg.y, msg.theta]

    def goal_callback(self, msg):
        self.current_goal = [msg.x, msg.y]

    def control_loop(self):
        if not self.enabled:
            self.pub.publish(Twist())
            return
        if self.current_pose is None or self.current_goal is None:
            return
        Kv = self.get_parameter('Kv').value
        Kw = self.get_parameter('Kw').value
        tolerance = self.get_parameter('tolerance').value
        v, w, d = compute_go_to_goal_control(
            self.current_pose, self.current_goal, Kv=Kv, Kw=Kw
        )
        cmd = Twist()
        if d < tolerance:
            self.pub.publish(cmd)
            return
        cmd.linear.x = v
        cmd.angular.z = w
        self.pub.publish(cmd)

    def validate_params(self, params):
        for p in params:
            if p.value <= 0:
                return SetParametersResult(
                    successful=False,
                    reason=f'{p.name} must be positive'
                )
        return SetParametersResult(successful=True)


def main():
    parser = argparse.ArgumentParser()
    parser.add_argument('--frequency', type=float, default=10.0)
    args, remaining = parser.parse_known_args()
    rclpy.init(args=remaining)
    node = TurtleController(frequency=args.frequency)
    try:
        rclpy.spin(node)
    except KeyboardInterrupt:
        pass
    finally:
        node.destroy_node()
        if rclpy.ok():
            rclpy.shutdown()


if __name__ == '__main__':
    main()
```

## Answer Key

```python
import argparse

import rclpy
from rclpy.lifecycle import LifecycleNode, TransitionCallbackReturn
from rclpy.qos import QoSPresetProfiles
from geometry_msgs.msg import Twist, Point
from turtlesim.msg import Pose
from rcl_interfaces.msg import SetParametersResult
from turtlesim_controller.control_law import compute_go_to_goal_control


class TurtleLfController(LifecycleNode):
    def __init__(self, frequency=10.0):
        super().__init__('turtle_lf_controller')
        self.frequency = frequency
        self.current_pose = None
        self.current_goal = None
        self.enabled = False
        self.pub = None
        self.declare_parameter('Kv', 1.0)
        self.declare_parameter('Kw', 5.0)
        self.declare_parameter('tolerance', 0.1)
        self.add_on_set_parameters_callback(self.validate_params)

    def on_configure(self, state):
        self.pub = self.create_publisher(Twist, 'turtle1/cmd_vel', 10)
        self.create_subscription(
            Pose, 'turtle1/pose', self.pose_callback,
            QoSPresetProfiles.SENSOR_DATA.value
        )
        self.create_subscription(Point, 'goal', self.goal_callback, 10)
        self.create_timer(1.0 / self.frequency, self.control_loop)
        self.get_logger().info('Configured.')
        return TransitionCallbackReturn.SUCCESS

    def on_activate(self, state):
        self.enabled = True
        self.get_logger().info('Activated ŌĆö control loop enabled.')
        return TransitionCallbackReturn.SUCCESS

    def on_deactivate(self, state):
        self.enabled = False
        if self.pub:
            self.pub.publish(Twist())
        self.get_logger().info('Deactivated ŌĆö published zero velocity.')
        return TransitionCallbackReturn.SUCCESS

    def on_shutdown(self, state):
        self.enabled = False
        if self.pub:
            self.pub.publish(Twist())
        return TransitionCallbackReturn.SUCCESS

    def pose_callback(self, msg):
        self.current_pose = [msg.x, msg.y, msg.theta]

    def goal_callback(self, msg):
        self.current_goal = [msg.x, msg.y]

    def control_loop(self):
        if not self.enabled:
            return
        if self.current_pose is None or self.current_goal is None:
            return
        Kv = self.get_parameter('Kv').value
        Kw = self.get_parameter('Kw').value
        tolerance = self.get_parameter('tolerance').value
        v, w, d = compute_go_to_goal_control(
            self.current_pose, self.current_goal, Kv=Kv, Kw=Kw
        )
        cmd = Twist()
        if d < tolerance:
            self.pub.publish(cmd)
            return
        cmd.linear.x = v
        cmd.angular.z = w
        self.pub.publish(cmd)

    def validate_params(self, params):
        for p in params:
            if p.value <= 0:
                return SetParametersResult(
                    successful=False,
                    reason=f'{p.name} must be positive'
                )
        return SetParametersResult(successful=True)


def main():
    parser = argparse.ArgumentParser()
    parser.add_argument('--frequency', type=float, default=10.0)
    args, remaining = parser.parse_known_args()
    rclpy.init(args=remaining)
    node = TurtleLfController(frequency=args.frequency)
    try:
        rclpy.spin(node)
    except KeyboardInterrupt:
        pass
    finally:
        node.destroy_node()
        if rclpy.ok():
            rclpy.shutdown()


if __name__ == '__main__':
    main()
```

`self.frequency` is stored as a plain attribute in `__init__` because the timer is created later in `on_configure` ŌĆö argparse delivers the value before spin starts, but it cannot be used until configure time.

The `controller_enabled` service is gone. The `deactivate` transition is a first-class lifecycle mechanism that replaces it ŌĆö no custom boolean service is needed. The original `control_loop` published a zero `Twist` when disabled to stop the turtle on every timer tick; in the lifecycle version, `on_deactivate` publishes zero once (explicitly, at the moment of deactivation), and the guard returns immediately without publishing on every subsequent tick. One deliberate stop is cleaner than repeated zero-velocity spam.

## Summary

| Concept | Key point |
|---|---|
| `LifecycleNode` | `from rclpy.lifecycle import LifecycleNode` ŌĆö replaces `Node`; adds state machine |
| `TransitionCallbackReturn.SUCCESS` | Confirms the transition completed; `.FAILURE` aborts and triggers error handling |
| `on_configure` | All `create_*` calls go here ŌĆö NOT in `__init__` |
| `on_activate` | Set the guard flag; fire async one-time setup calls |
| `on_deactivate` | Clear the flag; publish zero velocity explicitly |
| `on_shutdown` | Last-resort zero velocity ŌĆö called from any state |
| The guard pattern | `if not self.enabled: return` at the top of the timer callback |
| `call_async` in lifecycle callbacks | Replaces `spin_until_future_complete` ŌĆö spin is already running by the time callbacks fire |
| `ros2 lifecycle get /node` | Shows current state |
| `ros2 lifecycle set /node <transition>` | Drives a transition ŌĆö also done by launch files |
| `__init__` rule | Parameters and Python attribute init only ŌĆö no ROS entities |
