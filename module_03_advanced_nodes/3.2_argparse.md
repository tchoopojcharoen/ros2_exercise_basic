## Why Command-Line Arguments?

Look at `circle_publisher.py` from 2.5:

```python
def timer_callback(self):
    msg = Twist()
    msg.linear.x = 1.0
    msg.angular.z = 1.0
    self.pub.publish(msg)
```

The radius and speed are hardcoded. To change them, you edit the file and rebuild. That is fine for a one-off script but breaks down quickly — if you want to test several radii in a row, you rebuild each time, which is slow and loses the previous run's output.

The fix is standard Python `argparse`: accept values on the command line and pass them to the node at launch time. No edit, no rebuild.

---

## How argparse Coexists with `--ros-args`

ROS2 itself uses command-line arguments — `--ros-args` lets you remap topics and set parameters at launch time. This creates a conflict: `argparse.parse_args()` sees `--ros-args` as an unknown flag and exits with an error.

The fix is `parse_known_args()` instead of `parse_args()`:

```python
args, remaining = parser.parse_known_args()
```

`parse_known_args()` splits `sys.argv` into two lists:
- `args` — the flags your parser declared (`--radius`, `--speed`, etc.)
- `remaining` — everything else, including any `--ros-args` the user passed

You then hand `remaining` to `rclpy.init()`:

```python
rclpy.init(args=remaining)
```

Each side consumes only what it understands. Your custom flags never reach ROS2; `--ros-args` never reaches argparse.

### Passing arguments at launch

Use the `--` separator to tell `ros2 run` that what follows belongs to the executable, not to `ros2 run` itself:

```bash
ros2 run turtlesim_controller circle_publisher -- --radius 2.0 --speed 0.5
```

Without `--`, `ros2 run` would try to interpret `--radius` itself and fail.

## Step 1 — Modify `circle_publisher.py`

Open `~/tutorial_ws/src/turtlesim_controller/turtlesim_controller/circle_publisher.py`. You will make three changes to the existing file.

### Change 1 — Add the import

At the top of the file, add:

```python
import argparse
```

### Change 2 — Accept radius and speed in `__init__`

Replace the hardcoded constructor with one that takes `radius` and `speed` and computes the velocities from them. The relationship `v = r × w` gives `angular_z = speed / radius`.

Replace the current `__init__` with:

```python
def __init__(self, radius=1.0, speed=1.0):
    super().__init__('circle_publisher')
    self.linear_x = speed
    self.angular_z = speed / radius
    self.pub = self.create_publisher(Twist, 'turtle1/cmd_vel', 10)
    self.create_timer(0.1, self.timer_callback)
    self.get_logger().info(
        f'radius={radius:.2f} m  speed={speed:.2f} m/s  '
        f'angular_z={self.angular_z:.3f} rad/s'
    )
```

Update `timer_callback` to use the stored attributes instead of literals:

```python
def timer_callback(self):
    msg = Twist()
    msg.linear.x = self.linear_x
    msg.angular.z = self.angular_z
    self.pub.publish(msg)
```

### Change 3 — Replace `main()` with an argparse-aware version

The existing `main(args=None)` passes `args` directly to `rclpy.init()`. With argparse, we read from `sys.argv` instead — so the parameter goes away and we use `parse_known_args()`.

Replace `main()` with:

```python
def main():
    parser = argparse.ArgumentParser()
    parser.add_argument('--radius', type=float, default=1.0)
    parser.add_argument('--speed', type=float, default=1.0)
    args, remaining = parser.parse_known_args()

    rclpy.init(args=remaining)
    node = CirclePublisher(radius=args.radius, speed=args.speed)
    try:
        rclpy.spin(node)
    except KeyboardInterrupt:
        pass
    finally:
        node.destroy_node()
        if rclpy.ok():
            rclpy.shutdown()
```

The `if __name__ == '__main__': main()` guard at the bottom of the file stays unchanged — `main()` with no arguments still works.

### The complete modified file

After all three changes, `circle_publisher.py` should look like this:

```python
import argparse

import rclpy
from rclpy.node import Node
from geometry_msgs.msg import Twist


class CirclePublisher(Node):
    def __init__(self, radius=1.0, speed=1.0):
        super().__init__('circle_publisher')
        self.linear_x = speed
        self.angular_z = speed / radius
        self.pub = self.create_publisher(Twist, 'turtle1/cmd_vel', 10)
        self.create_timer(0.1, self.timer_callback)
        self.get_logger().info(
            f'radius={radius:.2f} m  speed={speed:.2f} m/s  '
            f'angular_z={self.angular_z:.3f} rad/s'
        )

    def timer_callback(self):
        msg = Twist()
        msg.linear.x = self.linear_x
        msg.angular.z = self.angular_z
        self.pub.publish(msg)


def main():
    parser = argparse.ArgumentParser()
    parser.add_argument('--radius', type=float, default=1.0)
    parser.add_argument('--speed', type=float, default=1.0)
    args, remaining = parser.parse_known_args()

    rclpy.init(args=remaining)
    node = CirclePublisher(radius=args.radius, speed=args.speed)
    try:
        rclpy.spin(node)
    except KeyboardInterrupt:
        pass
    finally:
        node.destroy_node()
        if rclpy.ok():
            rclpy.shutdown()


if __name__ == '__main__':
    main()
```

`setup.py` does not change — the `circle_publisher` entry point is already registered.

## The Limitation

Argparse arguments are read once — when the node starts. You cannot change `--radius` while the node is running. To do that, you would have to kill the node and restart it with new values.

This is inconvenient and impossible in some deployment scenarios. ROS2 parameters solve this by letting you read and write named values while the node is running. That is the topic of 3.3.

## Answer Key

```python
import argparse

import rclpy
from rclpy.node import Node
from geometry_msgs.msg import Twist, Point
from turtlesim.msg import Pose
from turtlesim_controller.control_law import compute_go_to_goal_control


class TurtleController(Node):
    def __init__(self, frequency=10.0):
        super().__init__('turtle_controller')
        self.current_pose = None
        self.current_goal = None
        self.pub = self.create_publisher(Twist, 'turtle1/cmd_vel', 10)
        self.create_subscription(Pose, 'turtle1/pose', self.pose_callback, 10)
        self.create_subscription(Point, 'goal', self.goal_callback, 10)
        self.create_timer(1.0 / frequency, self.control_loop)
        self.get_logger().info(f'Control loop frequency: {frequency} Hz')

    def pose_callback(self, msg):
        self.current_pose = [msg.x, msg.y, msg.theta]

    def goal_callback(self, msg):
        self.current_goal = [msg.x, msg.y]

    def control_loop(self):
        if self.current_pose is None or self.current_goal is None:
            return
        v, w, d = compute_go_to_goal_control(self.current_pose, self.current_goal)
        cmd = Twist()
        if d < 0.1:
            self.pub.publish(cmd)
            return
        cmd.linear.x = v
        cmd.angular.z = w
        self.pub.publish(cmd)


def main():
    parser = argparse.ArgumentParser()
    parser.add_argument('--frequency', type=float, default=10.0)
    args, remaining = parser.parse_known_args()

    rclpy.init(args=remaining)
    node = TurtleController(frequency=args.frequency)
    try:
        rclpy.spin(node)
    except KeyboardInterrupt:
        pass
    finally:
        node.destroy_node()
        if rclpy.ok():
            rclpy.shutdown()


if __name__ == '__main__':
    main()
```

The only structural change from 2.7 is that the hardcoded `0.1` in `create_timer` becomes `1.0 / frequency`, the `__init__` gains a `frequency` parameter with a default that preserves the original behaviour, and `main()` switches from `args=None` to the argparse pattern.

---

## Summary

| Concept | Key point |
|---|---|
| `argparse.ArgumentParser()` | Standard Python library — no additional dependencies |
| `parse_known_args()` | Returns `(known_args, remaining)` — remaining absorbs `--ros-args` without error |
| `rclpy.init(args=remaining)` | Passes ROS-specific flags to rclpy; your node never sees them |
| `-- --flag value` syntax | `--` tells `ros2 run` to stop consuming arguments and pass the rest to the executable |
| `main()` takes no parameter | argparse reads from `sys.argv` directly; no need for the `args=None` convention |
| Limitation | Values are fixed at launch; the node cannot update them while running |
