## Parameters vs Argparse

In 3.2, `--radius` and `--speed` were read once at startup and could not change while the node was running. If you wanted a different radius, you had to kill the node and restart it.

ROS2 parameters solve this. A parameter is a named value stored on the node that any other process can read or write while the node is running. The node reads the current value on every timer tick so changes take effect immediately — no restart.

The trade-off is a little more boilerplate in exchange for runtime tunability.

---

## How Parameters Work

### Declaring a Parameter

Call `declare_parameter(name, default_value)` in `__init__` to register a parameter. The default is used until someone sets a different value.

```python
self.declare_parameter('radius', 1.0)
self.declare_parameter('angular_speed', 1.0)
```

The type of the default value determines the parameter's type. A `float` default creates a float parameter.

### Reading a Parameter

Call `get_parameter(name).value` to read the current value. Call this inside a callback, not once in `__init__`, so the node picks up changes without restarting.

```python
def timer_callback(self):
    radius = self.get_parameter('radius').value
    w = self.get_parameter('angular_speed').value
    ...
```

If you read the value in `__init__` and store it (e.g. `self.radius = ...`), that copy is frozen at construction time and will not reflect later `param set` calls.

### Validating Changes

Register a callback that runs before any parameter change is applied:

```python
self.add_on_set_parameters_callback(self.validate_params)
```

The callback receives a list of `Parameter` objects. Return `SetParametersResult(successful=True)` to accept, or `SetParametersResult(successful=False, reason='...')` to reject.

```python
from rcl_interfaces.msg import SetParametersResult

def validate_params(self, params):
    for p in params:
        if p.name in ('radius', 'angular_speed') and p.value <= 0:
            return SetParametersResult(
                successful=False,
                reason=f'{p.name} must be positive'
            )
    return SetParametersResult(successful=True)
```

If rejected, the node keeps the previous value.

## Step 1 — Modify `circle_publisher.py`

Open `~/tutorial_ws/src/turtlesim_controller/turtlesim_controller/circle_publisher.py`. You will replace the argparse approach from 3.2 with ROS2 parameters.

### Change 1 — Replace the import

Remove `import argparse` and add the parameter result type:

```python
from rcl_interfaces.msg import SetParametersResult
```

### Change 2 — Replace `__init__`

The argparse version stored `self.linear_x` and `self.angular_z` at construction time, computed from the command-line values. With parameters, `__init__` only declares the parameters — the timer callback reads them fresh on every tick.

Replace the current `__init__` with:

```python
def __init__(self):
    super().__init__('circle_publisher')
    self.declare_parameter('radius', 1.0)
    self.declare_parameter('angular_speed', 1.0)
    self.pub = self.create_publisher(Twist, 'turtle1/cmd_vel', 10)
    self.create_timer(0.1, self.timer_callback)
    self.add_on_set_parameters_callback(self.validate_params)
```

### Change 3 — Update `timer_callback`

Replace the stored-attribute reads with live parameter reads. The relationship is `v = r × w`, so `linear.x = radius × angular_speed`:

```python
def timer_callback(self):
    radius = self.get_parameter('radius').value
    w = self.get_parameter('angular_speed').value
    msg = Twist()
    msg.linear.x = radius * w
    msg.angular.z = w
    self.pub.publish(msg)
```

### Change 4 — Add `validate_params`

Add a new method to reject invalid values:

```python
def validate_params(self, params):
    for p in params:
        if p.name in ('radius', 'angular_speed') and p.value <= 0:
            return SetParametersResult(
                successful=False,
                reason=f'{p.name} must be positive'

    return SetParametersResult(successful=True)
```

### Change 5 — Restore `main(args=None)`

Argparse read from `sys.argv` directly, so `main()` took no argument. Parameters use the standard ROS2 init convention — restore the `args=None` parameter and remove the `parse_known_args()` call:

```python
def main(args=None):
```

### The complete modified file

After all five changes, `circle_publisher.py` should look like this:

```python
import rclpy
from rclpy.node import Node
from geometry_msgs.msg import Twist
from rcl_interfaces.msg import SetParametersResult


class CirclePublisher(Node):
    def __init__(self):
        super().__init__('circle_publisher')
        self.declare_parameter('radius', 1.0)
        self.declare_parameter('angular_speed', 1.0)
        self.pub = self.create_publisher(Twist, 'turtle1/cmd_vel', 10)
        self.create_timer(0.1, self.timer_callback)
        self.add_on_set_parameters_callback(self.validate_params)

    def timer_callback(self):
        radius = self.get_parameter('radius').value
        w = self.get_parameter('angular_speed').value
        msg = Twist()
        msg.linear.x = radius * w
        msg.angular.z = w
        self.pub.publish(msg)

    def validate_params(self, params):
        for p in params:
            if p.name in ('radius', 'angular_speed') and p.value <= 0:
                return SetParametersResult(
                    successful=False,
                    reason=f'{p.name} must be positive'
                )
        return SetParametersResult(successful=True)


def main(args=None):
    rclpy.init(args=args)
    node = CirclePublisher()
    try:
        rclpy.spin(node)
    except KeyboardInterrupt:
        pass
    finally:
        node.destroy_node()
        if rclpy.ok():
            rclpy.shutdown()


if __name__ == '__main__':
    main()
```

## Step 5 — Live Tuning

With both turtlesim and `circle_publisher` running, change `radius` from any terminal:

```bash
ros2 param set /circle_publisher radius 3.0
```

The turtle immediately begins tracing a larger circle — no restart needed. On the next timer tick, `get_parameter('radius').value` returns `3.0`.

Try a smaller radius:

```bash

```

Now try to set an invalid value:

```bash
ros2 param set /circle_publisher angular_speed -1.0
```

The node rejects the change:

```
Setting parameter failed: angular_speed must be positive
```

The turtle keeps moving at the previous angular speed.

## Exercise

Apply parameters to `turtle_controller.py`. This combines what you learned in the 3.2 exercise (`--frequency` via argparse) with what you learned here (runtime-tunable values via parameters).

The 3.2 exercise answer key — your starting point:

```python
import argparse

import rclpy
from rclpy.node import Node
from geometry_msgs.msg import Twist, Point
from turtlesim.msg import Pose
from turtlesim_controller.control_law import compute_go_to_goal_control


class TurtleController(Node):
    def __init__(self, frequency=10.0):
        super().__init__('turtle_controller')
        self.current_pose = None
        self.current_goal = None
        self.pub = self.create_publisher(Twist, 'turtle1/cmd_vel', 10)
        self.create_subscription(Pose, 'turtle1/pose', self.pose_callback, 10)
        self.create_subscription(Point, 'goal', self.goal_callback, 10)
        self.create_timer(1.0 / frequency, self.control_loop)
        self.get_logger().info(f'Control loop frequency: {frequency} Hz')

    def pose_callback(self, msg):
        self.current_pose = [msg.x, msg.y, msg.theta]

    def goal_callback(self, msg):
        self.current_goal = [msg.x, msg.y]

    def control_loop(self):
        if self.current_pose is None or self.current_goal is None:
            return
        v, w, d = compute_go_to_goal_control(self.current_pose, self.current_goal)
        cmd = Twist()
        if d < 0.1:
            self.pub.publish(cmd)
            return
        cmd.linear.x = v
        cmd.angular.z = w
        self.pub.publish(cmd)


def main():
    parser = argparse.ArgumentParser()
    parser.add_argument('--frequency', type=float, default=10.0)
    args, remaining = parser.parse_known_args()

    rclpy.init(args=remaining)
    node = TurtleController(frequency=args.frequency)
    try:
        rclpy.spin(node)
    except KeyboardInterrupt:
        pass
    finally:
        node.destroy_node()
        if rclpy.ok():
            rclpy.shutdown()


if __name__ == '__main__':
    main()
```

Add three ROS2 parameters to this node:

| Parameter | Default | Meaning |
|---|---|---|
| `Kv` | 1.0 | Linear velocity gain passed to `compute_go_to_goal_control` |
| `Kw` | 5.0 | Angular velocity gain passed to `compute_go_to_goal_control` |
| `tolerance` | 0.1 | Distance below which the turtle is considered at goal |

Keep `--frequency` as an argparse argument — the control loop rate is fixed at launch, which is fine. `Kv`, `Kw`, and `tolerance` are tunable while running so you can watch the turtle's behaviour change immediately.

Add a validation callback that rejects non-positive values for all three parameters.

## Answer Key

```python
import argparse

import rclpy
from rclpy.node import Node
from geometry_msgs.msg import Twist, Point
from turtlesim.msg import Pose
from rcl_interfaces.msg import SetParametersResult
from turtlesim_controller.control_law import compute_go_to_goal_control


class TurtleController(Node):
    def __init__(self, frequency=10.0):
        super().__init__('turtle_controller')
        self.current_pose = None
        self.current_goal = None
        self.declare_parameter('Kv', 1.0)
        self.declare_parameter('Kw', 5.0)
        self.declare_parameter('tolerance', 0.1)
        self.pub = self.create_publisher(Twist, 'turtle1/cmd_vel', 10)
        self.create_subscription(Pose, 'turtle1/pose', self.pose_callback, 10)
        self.create_subscription(Point, 'goal', self.goal_callback, 10)
        self.create_timer(1.0 / frequency, self.control_loop)
        self.add_on_set_parameters_callback(self.validate_params)
        self.get_logger().info(f'Control loop frequency: {frequency} Hz')

    def pose_callback(self, msg):
        self.current_pose = [msg.x, msg.y, msg.theta]

    def goal_callback(self, msg):
        self.current_goal = [msg.x, msg.y]

    def control_loop(self):
        if self.current_pose is None or self.current_goal is None:
            return
        Kv = self.get_parameter('Kv').value
        Kw = self.get_parameter('Kw').value
        tolerance = self.get_parameter('tolerance').value
        v, w, d = compute_go_to_goal_control(
            self.current_pose, self.current_goal, Kv=Kv, Kw=Kw
        )
        cmd = Twist()
        if d < tolerance:
            self.pub.publish(cmd)
            return
        cmd.linear.x = v
        cmd.angular.z = w
        self.pub.publish(cmd)

    def validate_params(self, params):
        for p in params:
            if p.value <= 0:
                return SetParametersResult(
                    successful=False,
                    reason=f'{p.name} must be positive'
                )
        return SetParametersResult(successful=True)


def main():
    parser = argparse.ArgumentParser()
    parser.add_argument('--frequency', type=float, default=10.0)
    args, remaining = parser.parse_known_args()

    rclpy.init(args=remaining)
    node = TurtleController(frequency=args.frequency)
    try:
        rclpy.spin(node)
    except KeyboardInterrupt:
        pass
    finally:
        node.destroy_node()
        if rclpy.ok():
            rclpy.shutdown()


if __name__ == '__main__':
    main()
```

`--frequency` stays as argparse because the timer period is structural — changing it while running would require cancelling and recreating the timer, which is more complexity than this exercise needs. `Kv`, `Kw`, and `tolerance` are pure numbers read on every tick, so no rebuild logic is needed; they just work.

## Summary

| Concept | Key point |
|---|---|
| `declare_parameter(name, default)` | Registers a parameter; type is inferred from the default value |
| `get_parameter(name).value` | Read current value — call this inside a callback to pick up live changes |
| `add_on_set_parameters_callback(cb)` | Called before any change is applied; return `SetParametersResult` to accept or reject |
| `SetParametersResult(successful=False, reason=...)` | Rejects the change; node keeps the previous value |
| `ros2 param list /node` | List all parameters on a node |
| `ros2 param get /node name` | Read a single parameter's current value |
| `ros2 param set /node name value` | Set a parameter at runtime — no restart needed |
| Argparse vs parameters | Argparse: set once at launch. Parameters: readable and settable at any time |
