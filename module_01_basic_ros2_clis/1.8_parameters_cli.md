## What Are Parameters?

From 1.3: a parameter is a named configuration value on a specific node, readable and writable at runtime without restarting the node.

Parameters are node-local — `/turtlesim`'s `background_r` is a different parameter from any `background_r` on any other node. They are typed (integer, float, string, boolean, or arrays of these) and can have constraints such as minimum and maximum values.

Parameters are distinct from topics and services:
- A topic streams data continuously between nodes
- A service performs an operation on request
- A parameter configures how a node behaves — it is a setting, not a communication channel

## `ros2 param list`

Lists all parameters currently declared on a node.

```
ros2 param list /turtlesim
```

Expected output:

```
/turtlesim:
  background_b
  background_g
  background_r
  use_sim_time
```

The three `background_*` parameters control the RGB colour of the simulator window. `use_sim_time` is a standard parameter that every node has — it controls whether the node uses wall-clock time or a simulated clock.

To list parameters across all running nodes at once, omit the node name:

```
ros2 param list
```

---

## `ros2 param get`

Reads the current value of a single parameter.

```
ros2 param get /turtlesim background_r
```

```
ros2 param get /turtlesim background_g
```

```
ros2 param get /turtlesim background_b
```

The default turtlesim background is a blue-green colour. The default values are approximately `r=69, g=86, b=255`, but the exact values depend on the turtlesim version.

---

## `ros2 param set`

Writes a new value to a parameter at runtime. The node does not restart — the change takes effect immediately.

Change the background colour to red:

```
ros2 param set /turtlesim background_r 255
```

```
ros2 param set /turtlesim background_g 0
```

```
ros2 param set /turtlesim background_b 0
```

The simulator window background should turn red immediately — no restart required.

Restore the original colour:

```
ros2 param set /turtlesim background_r 69
```

```
ros2 param set /turtlesim background_g 86
```

```
ros2 param set /turtlesim background_b 255
```

---

## `ros2 param describe`

Shows a parameter's type, description, and any constraints (minimum, maximum, step).

```
ros2 param describe /turtlesim background_r
```

Expected output:

```
Parameter name: background_r
  Type: integer
  Description: Red channel of the background color
  Constraints:
    Min value: 0
    Max value: 255
    Step: 1
```

Constraints are declared by the node author. If you attempt to set a value outside the allowed range, the node will reject it. Not all parameters have constraints — `use_sim_time` is a boolean with no range.

---

## `ros2 param dump`

Exports all of a node's current parameter values to YAML. This is useful for saving a configuration to reload later.

```
ros2 param dump /turtlesim
```

Expected output:

```yaml
/turtlesim:
  ros__parameters:
    background_b: 255
    background_g: 86
    background_r: 69
    use_sim_time: false
```

To save this to a file instead of printing it:

```bash
ros2 param dump /turtlesim --output-dir ~/
```

This writes `~/turtlesim.yaml`. The file can be loaded back at startup with:

```bash
ros2 run turtlesim turtlesim_node --ros-args --params-file ~/turtlesim.yaml
```

Or applied to an already-running node with `ros2 param load`:

```bash
ros2 param load /turtlesim ~/turtlesim.yaml
```

YAML parameter files become important in later modules when configuring nodes through launch files.
