## What Is a Package?

A **package** is the unit of software distribution in ROS2. It is a directory inside `src/` that ROS2 and colcon can discover, build, and install. All the nodes you write will live inside packages.

A workspace can contain many packages, and each one is built and installed independently. This tutorial uses **`ament_python`** packages — pure Python, no C++ compiler involved.

---

## Step 1 — Create the Package

Navigate into the workspace `src/` directory and create a new Python package called `my_first_pkg`:

```
cd ~/tutorial_ws/src
```

```
ros2 pkg create --build-type ament_python my_first_pkg --dependencies rclpy std_msgs
```

Breaking down the flags:

| Flag | Meaning |
|---|---|
| `--build-type ament_python` | This is a pure Python package; colcon will not run a C++ compiler |
| `--dependencies rclpy std_msgs` | Declares runtime dependencies; colcon adds them to `package.xml` automatically |
| `my_first_pkg` | The package name — also becomes the directory name and Python module name |

You should see output like:

```
going to create a new package
package name: my_first_pkg
...
creating  ./my_first_pkg/package.xml
creating  ./my_first_pkg/setup.py
creating  ./my_first_pkg/setup.cfg
creating  ./my_first_pkg/my_first_pkg/__init__.py
```

---

## What Was Generated?

Inspect the package directory:

```
ls -R ~/tutorial_ws/src/my_first_pkg
```

Expected output:

```
/home/<user>/tutorial_ws/src/my_first_pkg:
package.xml  resource  setup.cfg  setup.py  my_first_pkg

/home/<user>/tutorial_ws/src/my_first_pkg/resource:
my_first_pkg

/home/<user>/tutorial_ws/src/my_first_pkg/my_first_pkg:
__init__.py
```

Notice the two levels named `my_first_pkg`:

| Path | What it is |
|---|---|
| `src/my_first_pkg/` | The ROS2 package root — contains `package.xml`, `setup.py`, etc. |
| `src/my_first_pkg/my_first_pkg/` | The Python module — this is where your `.py` node files go |

All node scripts you write will be placed inside the inner `my_first_pkg/` directory. You do not need to understand `package.xml` or `setup.py` yet — you will explore them in 2.3 when you add your first node.

---

## Step 2 — Build the Package

Build only `my_first_pkg` using `--packages-select`. This is faster than rebuilding the entire workspace when you only changed one package:

```
cd ~/tutorial_ws
```

```
colcon build --packages-select my_first_pkg
```

Expected output:

```
Starting >>> my_first_pkg
Finished <<< my_first_pkg [1.2s]

Summary: 1 package finished [1.5s]
```

Now re-source the overlay so this terminal sees the newly installed package:

```
source ~/tutorial_ws/install/setup.bash
```

---

## Step 3 — Verify the Package Is Installed

Confirm that ROS2 can find `my_first_pkg`:

```
ros2 pkg list | grep my_first_pkg
```

Expected output:

```
my_first_pkg
```

If the package does not appear, the overlay was not re-sourced after the build. Run `source ~/tutorial_ws/install/setup.bash` and try again.

---

## Summary

| Concept | Key point |
|---|---|
| Package | A directory in `src/` that colcon can build and ROS2 can discover |
| `ament_python` | The build type for pure Python packages |
| `--dependencies` | Declares runtime dependencies; added to `package.xml` automatically |
| `--packages-select` | Builds only the named package — faster than rebuilding everything |
| Re-source after build | Required in any open terminal after `colcon build` changes the overlay |
