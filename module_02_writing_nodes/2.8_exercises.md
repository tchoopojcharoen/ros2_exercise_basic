## System Overview

This exercise runs three nodes simultaneously:

| Node | Package | What it does |
|---|---|---|
| `turtlesim_node` | `turtlesim` | Simulates the turtle; publishes `/turtle1/pose`, accepts `/turtle1/cmd_vel` |
| `goal_point_publisher` | `goal_point_publisher` | Opens a GUI window; clicking sets a goal; publishes `geometry_msgs/msg/Point` on `/goal` |
| `turtle_controller` | `turtlesim_controller` | **You write this** — reads pose and goal, computes velocity commands, publishes to `/turtle1/cmd_vel` |

The data flow:

```
turtlesim_node ──► turtle1/pose ──────────────► turtle_controller
goal_point_publisher ──────────► goal ────────► turtle_controller
                                                       │
                                                       ▼
                               turtle1/cmd_vel ──► turtlesim_node
```

`turtle_controller` is the only node you write. The other two already exist.

## Answer Key

```python
import rclpy
from rclpy.node import Node
from geometry_msgs.msg import Twist, Point
from turtlesim.msg import Pose
from rclpy.qos import QoSPresetProfiles
from turtlesim_controller.control_law import compute_go_to_goal_control


class TurtleController(Node):
    def __init__(self):
        super().__init__('turtle_controller')
        self.current_pose = None
        self.current_goal = None
        self.pub = self.create_publisher(Twist, '/turtle1/cmd_vel', 10)
        self.create_subscription(
            Pose, '/turtle1/pose', self.pose_callback,
            QoSPresetProfiles.SENSOR_DATA.value
        )
        self.create_subscription(Point, 'goal', self.goal_callback, 10)
        self.create_timer(0.1, self.control_loop)

    def pose_callback(self, msg: Pose):
        self.current_pose = [msg.x, msg.y, msg.theta]

    def goal_callback(self, msg: Point):
        self.current_goal = [msg.x, msg.y]

    def control_loop(self):
        if self.current_pose is None or self.current_goal is None:
            return
        v, w, d = compute_go_to_goal_control(self.current_pose, self.current_goal)
        cmd = Twist()
        if d < 0.1:
            self.pub.publish(cmd)
            return
        cmd.linear.x = v
        cmd.angular.z = w
        self.pub.publish(cmd)


def main(args=None):
    rclpy.init(args=args)
    node = TurtleController()
    try:
        rclpy.spin(node)
    except KeyboardInterrupt:
        pass
    finally:
        node.destroy_node()
        if rclpy.ok():
            rclpy.shutdown()


if __name__ == '__main__':
    main()
```

Two subscriptions and one publisher coexist on the same node object. The pose callback stores `[msg.x, msg.y, msg.theta]`; the goal callback stores `[msg.x, msg.y]` — `Point.z` is ignored. Both convert the incoming ROS message to a plain Python list to match the control law's API.

In `control_loop`, a zero `Twist()` is published when the turtle reaches the goal. Sending no command at all would leave the last non-zero velocity active, causing the turtle to drift past the goal.

The timer (10 Hz) reads `current_pose` and `current_goal`, which are written by the subscriber callbacks (~60 Hz for pose, occasional for goal). This is the same subscriber-stores / timer-publishes pattern from 2.6, extended to two independent data sources feeding one timer-based action.

---

## Summary

| Concept | Key point |
|---|---|
| Multiple subscriptions in one node | Call `create_subscription` once per topic in `__init__`; each gets its own callback |
| Subscriber-stores / timer-publishes | Callbacks store latest data; a separate timer reads and acts at a fixed rate |
| `None` guard with two attributes | `if self.current_pose is None or self.current_goal is None: return` — wait for both |
| Converting messages to plain types | `[msg.x, msg.y, msg.theta]` — pass lists, not ROS message objects, to non-ROS functions |
| Zero Twist on goal reached | Publish `Twist()` (all zeros) explicitly; do not rely on the absence of a command |
| Internal module import | `from turtlesim_controller.control_law import ...` — same package, no `<depend>` needed |
