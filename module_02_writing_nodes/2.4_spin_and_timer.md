## Timers and the Event Loop

In 2.3 you saw that `rclpy.spin()` runs an event loop — checking for pending work and dispatching it. A **timer** is one source of that work.

When you call `self.create_timer(period_sec, callback)`, you register a callback method with spin. Every `period_sec` seconds, spin marks that timer as due and calls your method. The relationship is:

```
create_timer(1.0, self.timer_callback)
          ↓
    spin loop ticks
          ↓
  every 1.0 s → timer_callback() is called
```

Your callback is a plain Python method — it takes no special arguments and returns nothing. The loop, the timing, and the dispatch are all handled by spin.

## Step 1 — Write the Node

Create `logger_node.py` inside `~/tutorial_ws/src/turtlesim_controller/turtlesim_controller/` and paste in the following:

```
import rclpy
from rclpy.node import Node


class LoggerNode(Node):
    def __init__(self):
        super().__init__('logger_node')
        self.create_timer(1.0, self.timer_callback)

    def timer_callback(self):
        self.get_logger().info('Hello from logger_node!')


def main(args=None):
    rclpy.init(args=args)
    node = LoggerNode()
    try:
        rclpy.spin(node)
    except KeyboardInterrupt:
        pass
    finally:
        node.destroy_node()
        if rclpy.ok():
            rclpy.shutdown()


if __name__ == '__main__':
    main()
```

### What is new compared to 2.3

| Addition | What it does |
|---|---|
| `self.create_timer(1.0, self.timer_callback)` | Schedules `timer_callback` to fire every 1.0 second via the spin loop |
| `def timer_callback(self)` | A plain method — spin calls this whenever the timer is due; no special arguments |

The callback itself does one thing: log a fixed string. There is no state — every call is identical. `self.create_timer` is called inside `__init__`, so the timer is registered as soon as the node is created. The first tick fires approximately 1.0 second after `rclpy.spin()` begins.

## Log Levels

`self.get_logger()` provides four severity levels:

```python
self.get_logger().debug('Verbose detail — hidden by default at runtime')
self.get_logger().info('Normal operation — shown by default')
self.get_logger().warn('Unexpected but recoverable')
self.get_logger().error('Something went wrong')
```

| Level | When to use |
|---|---|
| `debug` | Detailed diagnostic data — disabled by default to reduce noise |
| `info` | Routine events — node started, goal reached, configuration loaded |
| `warn` | Recoverable anomalies — missing optional data, approaching a limit |
| `error` | Failures that affect behaviour — required topic unavailable, bad input |

Use `info` for anything that confirms the node is working correctly. Reserve `warn` and `error` for conditions that require attention.

## Answer Key

```
import rclpy
from rclpy.node import Node


class LoggerNode(Node):
    def __init__(self):
        super().__init__('logger_node')
        self.count = 0
        self.create_timer(1.0, self.timer_callback)

    def timer_callback(self):
        self.count += 1
        self.get_logger().info(f'Tick #{self.count}')


def main(args=None):
    rclpy.init(args=args)
    node = LoggerNode()
    try:
        rclpy.spin(node)
    except KeyboardInterrupt:
        pass
    finally:
        node.destroy_node()
        if rclpy.ok():
            rclpy.shutdown()


if __name__ == '__main__':
    main()
```

`self.count` is initialised to `0` in `__init__` and incremented in every callback call. Because it is an instance variable on the node object, it persists between calls — spin calls the callback repeatedly, and the object retains its state throughout.

---

## Summary

| Concept | Key point |
|---|---|
| `create_timer(period, callback)` | Registers a callback to fire every `period` seconds inside the spin loop |
| Timer callback signature | Always `def callback(self):` — spin passes no arguments |
| Instance variables | Persist across callback calls; initialise them in `__init__` |
| `get_logger().info(...)` | Logs at INFO level; use `debug`, `warn`, `error` for other severities |
